Methodology & data sources

This page documents how leads in the dashboard are produced. The intent is auditability — every signal a lead carries is traceable to a public-records source, a date, and a join rule. No skip trace, no AVM, no MLS, no proprietary scoring.

The 6-pattern stack

A lead's tier is the count of distinct distress patterns that fire on its parcel. Score sums never determine tier — see Two-Truths invariant. Six orthogonal patterns:

PatternFires onNH-specific notes
jfc
Judicial Foreclosure		 StarNews foreclosure-sale notice (last 90 days), NHC active GS 105-374 schedule, ROD FCL/N/F/SUB TR/FCL DEED in last 18 months NC is a power-of-sale state. eCourts (Tyler Odyssey) is AWS-WAF-CAPTCHA-gated, so newspaper notice flow + ROD recordings are the parallel feed.
tax
Tax distress		 NHC delinquent-tax CSV with total due ≥ $500, parcel on the active GS 105-374 schedule The county delinquent CSV is downloadable directly — 5,120 rows, monthly refresh, parcel-level. Rare among NC counties; this is the highest-volume signal.
estate
Probate / Estate		 StarNews Notice to Creditors (decedent → owner-name match), ROD estate deeds (ADMIN DEED, EXEC DEED, EXTRX DEED, COMMR DEED) Decedent-to-parcel join is fuzzy — uses last+first+middle prefix matching with a top-50-surnames guard to prevent false-positive explosion.
code
Code / Demolition		 NHC EnerGov building permit where PERMIT_TYPE LIKE '%Demolition%' in last 24 months Documented gap: NH publishes no code-enforcement case dataset. Wilmington's EnerGov SelfService is gated behind a Tyler SPA + CSRF token (Phase 2+ Playwright). The code pattern in this build is structurally weaker than Mecklenburg's Charlotte HNS feed.
lien
Recorded lien / judgment		 ROD JDGMT, LIEN, LIS PENS, BKTCY recorded in last 24 months NHC ROD has no separate codes for federal/state tax liens, mechanics liens, or HOA liens — they all file under the generic LIEN code with the type in the description. We regex-extract the sub-type into doc_types.
transfer
Distressed conveyance		 ROD QCD (quitclaim) in last 24 months, ROD DEED OF GIFT, ROD separation-agreement co-signal, PropertyOwners-derived nominal-consideration recent sale (price < 5% of assessed value), or estate notice posted within 18 months prior to a parcel sale PropertyOwners-derived rules don't depend on ROD coverage. Even when ROD is unavailable, the nominal-sale and post-estate rules can fire from the parcel master alone.

Tier from stack depth

Stack depth is the count of distinct patterns matched. Tier comes from stack depth — never from raw score sum. Sub-flags adjust raw score for in-tier sorting but never promote tier.

  • Hot — stack_count ≥ 3
  • Warm — stack_count == 2
  • Active — stack_count == 1
  • Records with zero matched patterns are dropped from leads.json.

The Two-Truths invariant

The pipeline calls two_truths_check() immediately before writing leads.json. It recomputes tier_counts and pattern_counts from records[] and compares to the header values. Any mismatch raises and the file is not written. This guards against the most common dashboard failure mode: counts and visible records computed from different code paths drifting.

The dashboard reuses the same pattern + flag fields for both filter counts and the rendered table — there is one matches(lead) function in index.html and it powers both the stat-row totals and the table.

Sources NOT used (and why)

  • eCourts (Tyler Odyssey) — AWS WAF CAPTCHA every 5 minutes for both anonymous and registered users. Tyler does not provide a bulk feed. Pivoted to StarNews newspaper notices for the SP foreclosure / E estate signals (NC statute requires statutory publication anyway).
  • Wilmington EnerGov SelfService (city-issued permits + city code cases) — Tyler SPA + CSRF token + session cookie. Mid-effort Playwright scrape, deferred to Phase 2+.
  • Beach towns (Wrightsville Beach, Carolina Beach, Kure Beach) — no public open-data portals. Town-issued permits are not in any open feed.
  • Munis Self Service — per-parcel only, behind a Tyler Portico OAuth + AntiXsrfToken gate. The county's delinquent CSV is strictly superior — all 5,120 parcels in one file vs Munis' one-at-a-time UI.
  • Kania Law Firm + RBCWB — neither services New Hanover. The Mecklenburg dual-firm pattern does not transfer; NHC handles GS 105-374 cases in-house through the Tax Department + Clerk of Superior Court.
  • etax (iasWorld) — used as the dashboard's "open in county" deep-link target only. Not scraped for primary data; PropertyOwners feed already carries everything.

Join strategy

Every signal is attached to a parcel master record (PARID, format R#####-###-###-###) using the strongest available join:

  • Direct PARID: delinquent tax CSV, NHC foreclosures HTML, EnerGov permits, stormwater. These all carry the parcel ID directly.
  • Decedent-name join: StarNews Notice to Creditors. Decedent name is normalized and matched against owner_keys built from OWN1. A top-50-surnames guard prevents JONES JOHN from joining hundreds of parcels.
  • Address-key join: StarNews foreclosure body — regex extracts likely address strings, normalizes street name + suffix, joins to (street_num, street_name) index over PropertyOwners.
  • Reverse-party owner-name join: Register of Deeds — NHC ROD records carry no parcel ID. Reverse-party name is matched against owner_keys with two filters: governmental plaintiffs (CITY OF, DEPARTMENT OF, UNITED STATES, etc.) and corporate entities (LLC, INC, etc.) are skipped — they would explode to thousands of false-positive parcels.

What's not included (and why)

  • No skip trace — phone / email columns are placeholders for downstream skip-trace vendors.
  • No AVM, MLS pricing, mortgage balance, lender data — out of scope; assessed value (APRTOT) is the only value baseline.
  • No year-built — NHC PropertyOwners feed doesn't expose it. Would require per-parcel etax Datalet scrape; deferred.
  • No exemption flags (homestead / senior / disabled veteran) — same reason.
  • No ML scoring — every signal is rule-driven and explainable. raw_score is for in-tier sorting only and never promotes tier.

Daily refresh

The pipeline is built for daily live data. pipeline/refresh.py runs every scraper in dependency order, regenerates leads.json, runs the Two-Truths check, and pushes to GitHub. Windows Task Scheduler (config in scripts/daily_refresh.xml) runs it at 4 AM Central.

The header generated_at on the dashboard is the timestamp of the most recent successful pipeline run. HEARTBEAT.json in the repo root tracks per-source last_success for operations alerting.

Anti-bot encounters honestly

  • NHC ArcGIS, ROD BIS PHP, StarNews Gannett iPublish, NHC delinquent CSV, NHC foreclosures HTML — all open. No CAPTCHA, no Cloudflare WAF, no login wall.
  • eCourts (Tyler Odyssey) — AWS WAF CAPTCHA, blocked.
  • Wilmington EnerGov SelfService — Tyler SPA + CSRF token, requires Playwright.
  • Munis Self Service — Tyler Portico OAuth + AntiXsrfToken, redundant with the CSV anyway.
  • iasWorld etax — disclaimer-accept POST + viewstate, used as deep-link target only.
